Рекомендации по устранению выявленных ошибок и улучшению системы

1. Внедрение аутентификации и авторизации
Проблема: отсутствует механизм аутентификации пользователей на уровне API.

Решение: реализовать JWT‑аутентификацию:
создать эндпоинты /login и /register;
при успешной аутентификации выдавать JWT‑токен;
добавлять токен в заголовок Authorization для защищённых запросов;
проверять токен на бэкенде перед выполнением операций.

2. Миграция на промышленную СУБД
Проблема: использование SQLite, не предназначенного для продакшена.

Решение: перейти на PostgreSQL или MySQL:
настроить подключение к новой БД в приложении;
перенести существующие данные с помощью скриптов миграции;
обновить SQL‑запросы с учётом синтаксиса новой СУБД;
настроить резервное копирование и мониторинг БД.

3. Реализация пагинации
Проблема: отсутствие пагинации для больших наборов данных.

Решение: добавить пагинацию для списков постов и комментариев:
модифицировать эндпоинты /posts/public и /post/<id>/comments, добавив параметры page и limit;
на фронтенде добавить кнопки «Предыдущая/Следующая страница» или нумерацию страниц;
реализовать отображение общего количества элементов и текущей страницы.

4. Нормализация структуры тегов
Проблема: теги хранятся в виде строки, что затрудняет поиск и фильтрацию.

Решение: создать нормализованную схему:
таблица tags (id, name);
таблица связей post_tags (post_id, tag_id);
обновить бэкенд‑логику для работы с новой структурой;
добавить API‑эндпоинты для управления тегами (создание, поиск, удаление).

5. Улучшение фронтенда
Проблема: фронтенд без фреймворка, сложно масштабировать.

Решение: переписать интерфейс с использованием React/Vue.js:
разбить интерфейс на компоненты (пост, комментарий, форма);
внедрить маршрутизацию (React Router / Vue Router);
использовать состояние приложения (Redux / Vuex) для управления данными;
применить CSS‑фреймворк (Tailwind CSS, Bootstrap) для стилизации.
